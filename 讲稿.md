# Mini C++ IDE Simulator V2 — 路演讲稿

---

## 一、开场白（1 分钟）

各位老师、同学，大家好！

今天我为大家带来的项目是 **Mini C++ IDE Simulator V2**——一款**教学级 C++ 代码执行可视化调试器**。

简单来说，它就是一个**看得见的内存实验室**。你写一段 C++ 代码，它不仅能运行，还能把**每一行代码执行时内存里发生了什么**用图形界面展示出来——变量存在哪个地址、占多少字节、值怎么变化，一目了然。

它的目标人群是**正在学习 C++ 的初学者**。很多同学在学指针、学 vector、学内存分配时觉得抽象，就是因为**看不到内存里实际发生了什么**。我们这个工具就是要解决这个问题。

---

## 二、项目背景与动机（1 分钟）

传统的 C++ 学习路径是这样的：

- 写代码 → 编译 → 运行 → 看控制台输出

但这里有一个巨大的**黑箱**：

- 变量存在哪里？
- `vector.push_back` 之后内存怎么变的？
- 为什么 `int` 占 4 个字节而 `double` 占 8 个？
- `map<string,int>` 里面到底是什么结构？

这些对于初学者来说，光靠想象很难理解。市面上虽然有专业调试器（如 GDB、LLDB），但它们的界面复杂、学习曲线陡峭，不适合教学场景。

所以我们做了这个工具——**专门为教学设计的可视化调试器**。它运行在 Qt 图形界面中，把"编译执行"变成了**每一步都能看、能点、能回放**的交互体验。

## 三、整体架构与实现原理（3 分钟）

整个项目可以分为四个层次：

### 第一层：词法分析 —— Lexer（把代码变成单词）

**文件：** `src/lexer.h` / `src/lexer.cpp`

当用户在编辑器中输入代码，首先要做的事情是**分词**。

比如这段代码：
```cpp
int x = 42;
```

Lexer 会把它拆成一系列 Token（令牌）：
```
INT  IDENTIFIER("x")  ASSIGN  INT_LITERAL(42)  SEMICOLON
```

我们支持近 **60 种 Token 类型**：关键字（`int`、`for`、`if`、`cout`）、运算符（`+`、`<<`、`++`）、字面量（整数、浮点数、字符串、字符）、分隔符等等。

Lexer 逐字符扫描源代码，跳过空白和注释（支持 `//` 和 `/* */`），遇到数字调 `readNumber()`，遇到字符串调 `readString()`，遇到标识符调 `readIdentifierOrKeyword()`——后者会查表判断是一个关键字还是一个普通变量名。

### 第二层：语法分析 —— Parser（把单词变成语法树）

**文件：** `src/parser.h` / `src/parser.cpp`

Token 只是一维的序列，Parser 要把它们组织成**抽象语法树（AST）**。

我们采用的是**递归下降解析**——简单、直观、手写、无需外部工具。每一种语法结构对应一个解析函数：

- `parseExpression()` — 表达式（支持完整的运算符优先级：逻辑或 → 逻辑与 → 相等 → 关系 → 加减 → 乘除 → 一元 → 后缀 → 基本）
- `parseIf()` — `if/else if/else`
- `parseWhile()` / `parseDoWhile()` — while 和 do-while 循环
- `parseFor()` — for 循环（包括 range-based for）
- `parseCout()` / `parseCin()` — `cout <<` 和 `cin >>`
- `parseTryCatch()` — `try/catch/throw`

**AST 节点定义在 `src/ast.h`**，共有约 30 种节点类型：

- **表达式**：LiteralExpr（字面量）、VarExpr（变量引用）、BinaryExpr（二元运算）、UnaryExpr（一元运算）、IndexExpr（下标访问）、MemberAccessExpr（成员访问）、CallExpr（函数调用）、CastExpr（类型转换）、NewExpr/DeleteExpr（动态内存）、NullptrExpr 等
- **语句**：DeclStmt（声明）、AssignStmt（赋值）、IfStmt、WhileStmt、ForStmt、BreakStmt、ContinueStmt、CoutStmt、CinStmt、ReturnStmt、TryCatchStmt、ThrowStmt 等
- **顶层结构**：Program（整个程序）、FunctionDef（函数定义）、ClassDef（类/结构体定义）、TemplateClassDef（模板类）

解析完成后，我们得到一棵 AST，比如 `int x = 42;` 会变成：
```
Program
 └── FunctionDef (main)
      └── BlockStmt
           └── DeclStmt (type="int", name="x")
                └── init = LiteralExpr(42)
```

### 第三层：解释执行 —— Executor（遍历 AST 并执行）

**文件：** `src/executor.h` / `src/executor.cpp`

这是整个项目的**核心引擎**。它是一个树遍历解释器（tree-walking interpreter），遍历 AST 节点并执行。

Executor 内部维护：

- **符号表** `vars_`：`map<string, VariantValue>`，记录每个变量的当前值
- **类注册表** `classes_`：管理自定义 struct/class 的元信息
- **内存映射** `memoryMap_`：模拟内存地址，从 `0x1000` 开始分配
- **控制流状态**：break/continue 信号、异常栈（exceptionStack_）、try/catch 嵌套状态
- **快照管理器指针**：每执行一条语句就记录一次状态

执行流程：
1. 调用 `setAst()` 传入 Program
2. `runAll()` 或 `step()` 开始执行
3. 每次执行一个语句前，先把这个语句及其所有子语句展开到 `stepQueue_` 中
4. 执行完毕后调用 `takeSnapshot()` 记录当前状态
5. 如果是 step 模式，通过回调 `onYield_` 交出控制权给 GUI

**关键设计 —— 运行时类型系统（VariantValue）：**

**`src/variant_value.h`** 定义了 `VariantValue`，它是一个基于 `std::variant` 的**动态类型系统**。支持超过 **25 种类型**：

- 基本类型：`INT`、`FLOAT`、`DOUBLE`、`CHAR`、`BOOL`、`STRING`、`VOID`
- 容器类型：`VECTOR_INT`、`VECTOR_FLOAT`、`VECTOR_DOUBLE`、`VECTOR_CHAR`、`VECTOR_BOOL`、`VECTOR_STRING`、`LIST_INT`、`LIST_STRING`、`SET_INT`、`MAP_STRING_INT`
- 对象类型：`OBJECT`、`VECTOR_OBJECT`、`POINTER`、`NULL_PTR`

每个 VariantValue 提供：
- `typeName()` — 返回类型名（如 "vector<int>"）
- `toString()` — 返回值的字符串表示（用于显示在变量表中）
- `byteSize()` — 返回"模拟内存大小"（用于内存视图绘制）

### 第四层：快照系统与可视化（2 分钟）

**快照系统 —— `src/snapshot.h`**

每执行一行代码，Executor 调用 `takeSnapshot()` 记录一个 `Snapshot`，包含：

- 当前行号
- 所有变量的当前值
- 控制台输出内容
- 内存映射表

`SnapshotManager` 管理所有快照，支持通过 `currentIndex` 在快照之间跳转。这就是**时间轴回放功能**的基础。

**GUI 组件：**

1. **代码编辑器（CodeEditor）** — `src/codeeditor.h`
   - 基于 QPlainTextEdit，带行号显示
   - C++ 语法高亮（关键字、数字、字符串、注释、预处理指令）
   - **高亮当前执行行**——step 执行时，当前行被黄色高亮

2. **变量监视表（VarTable）**
   - QTableWidget，显示所有变量的**名、类型、值**
   - 值变化的行用橙色闪烁提示

3. **内存可视化（MemoryView）** — `src/memoryview.h`
   - 自定义 QWidget，用 QPainter 绘制
   - 每个变量显示为一个彩色方块，标注**地址、名称、值、类型**
   - 不同数据类型用不同颜色区分（int=蓝色, float=绿色, char=红色, string=紫色 等）
   - 地址从 0x1000 开始，按 byteSize 对齐

4. **控制台 I/O**
   - QPlainTextEdit 显示程序输出
   - QLineEdit 提供输入（cin 请求时自动弹出）

5. **控制面板**
   - **Run** — 连续执行至结束
   - **Step** — 单步执行一行
   - **Reset** — 重置状态
   - **AutoPlay** — 自动连续执行（速度可调）
   - **时间轴滑块** — 拖动滑块在任意快照间跳转

**整个数据流**：

```
用户输入代码
    ↓
Lexer 分词
    ↓
Parser 构建 AST
    ↓
Executor 遍历执行 AST  ←→ SnapshotManager 记录快照
    ↓                              ↓
GUI 更新显示              ←   读取快照数据
（编辑器高亮 + 变量表 + 内存视图 + 控制台）
```

## 六、技术挑战与解决方案（1 分钟）

### 挑战 1：动态类型系统
C++ 是静态类型语言，但我们的解释器需要在运行时处理各种类型。

**解决方案：** 基于 `std::variant` 实现 `VariantValue`，用 switch-case 分派不同类型操作。支持隐式类型提升（int → float → double）。

### 挑战 2：单步执行与状态保存
要让用户"点一下执行一行"，需要暂停执行并在每一步保存完整状态。

**解决方案：** 引入 `stepQueue_`（待执行语句队列）+ `SnapshotManager`（快照管理器）。每次执行前把复合语句展开为原子语句列表，执行一条就拍一张快照。

### 挑战 3：容器类型支持
vector、map、set 等容器的运行时要能动态增删元素。

**解决方案：** 利用 C++ 自身的 STL 作为 runtime 后端——`VariantValue` 内部直接存储 `std::vector<int>`、`std::map<string,int>` 等，解释器调用对应的 push_back、size 等方法。

### 挑战 4：字符串作为条件
C++ 中 `if (str)` 判断字符串是否非空。

**解决方案：** 在 Executor 的 `execIf`、`execWhile` 等方法中，对 STRING 类型的条件判断 `!value.empty()`。

## 附录：演示建议

### 推荐演示流程（约 8-10 分钟）

1. **启动程序**（30 秒）
   - 显示 GUI 界面，介绍各区域：代码编辑器、控制台、变量表、内存视图、控制按钮

2. **加载 demo_full.cpp**（30 秒）
   - 把 demo_full.cpp 粘贴到编辑器中

3. **运行完整程序**（1 分钟）
   - 点击 Run，展示全部 15 个部分的输出
   - 强调所有功能都能正确运行

4. **单步执行演示**（2 分钟）
   - Reset 后点击 Step，逐行执行
   - 观察代码行高亮移动
   - 观察变量表的数值变化
   - 观察内存视图的更新

5. **内存视图重点讲解**（2 分钟）
   - Step 到声明 `int i = 42;` 的下一行
   - 指出内存视图中变量 i 的地址（0x1000）、值（42）、类型（int，蓝色）
   - Step 到声明 `double d = 2.718;`，指出 double 占 8 字节（绿色）
   - Step 到 `vector<int> vi;` 和 `push_back` 之后，展示容器在内存中的布局

6. **时间轴回放**（1 分钟）
   - 执行几步后，拖动滑块往回拖
   - 展示变量表和内存视图恢复到之前的状态
   - 强调这是"时间旅行调试"

7. **如果时间允许：现场写一段简单代码**（1 分钟）
   - 例如：`int sum = 0; for (int i = 1; i <= 5; i++) { sum += i; } cout << sum << endl;`
   - 单步执行，展示循环过程中 sum 的累加变化

### 常见 QA 准备

- **Q: 这个和 GDB 有什么区别？**
  A: GDB 是专业级调试器，功能强大但界面复杂。我们的工具是专门为教学设计的，内存可视化直观、操作简单，适合 C++ 初学者理解内存模型。

- **Q: 支持所有 C++ 语法吗？**
  A: 不支持全部——我们聚焦于 C++ 入门课程的核心内容。不支持模板实例化、多重继承、lambda 表达式等高级特性。但支持了教学中最常用的所有语法。

- **Q: 地址是真实的吗？**
  A: 地址是模拟的，从 0x1000 开始分配。这样做是为了让初学者更容易理解内存布局的概念，而不会被真实操作系统复杂的内存管理（ASLR、分页等）搞混。

- **Q: 可以自己写代码跑吗？**
  A: 可以！直接在代码编辑器中写代码，点击 Run 或 Step 即可执行。注意使用我们支持的语法子集。
